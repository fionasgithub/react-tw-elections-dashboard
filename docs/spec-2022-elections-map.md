# 需求規格書：2022 縣市長選舉開票地圖

## 1. 概述

### 1.1 專案名稱

2022 縣市長選舉開票地圖

### 1.2 專案背景

提供一個以地圖為核心的視覺化介面，讓使用者能即時查看 2022 年台灣縣市長選舉各縣市及鄉鎮市區的開票結果。

### 1.3 技術範疇

- 前端框架：React + TypeScript
- 地圖繪製：D3.js + TopoJSON
- 樣式：Shadcn UI

---

## 2. 功能需求

### 2.1 頁面標題區塊

| 項目   | 說明                            |
| ------ | ------------------------------- |
| 主標題 | 顯示「2022 縣市長選舉開票地圖」 |

---

### 2.2 台灣縣市地圖（首頁主地圖）

#### 2.2.1 地圖渲染

- 使用 **D3.js** 搭配 **TopoJSON** 資料格式繪製台灣全島及離島縣市地圖
- 地圖單位為**縣市層級**（22 個縣市）
- 每個縣市色塊以**該縣市目前得票率最高政黨的代表色**填充（純色，不使用漸層）
- 開票前或尚無票數資料時，縣市色塊以 `EMPTY` 類別顯示初始淺灰色（`#D3D3D3`）
- 若最高票候選人出現**平票**或無資料，維持初始淺灰色，不顯示任何政黨代表色

#### 2.2.2 政黨代表色對照

以下政黨與代表色依專案實作（`src/types/elections.ts`）：

| 政黨代碼 | 政黨名稱      | 顏色      |
| -------- | ------------- | --------- |
| KMT      | 中國國民黨    | `#5B6FA6` |
| DPP      | 民主進步黨    | `#6FB98E` |
| TPP      | 台灣民眾黨    | `#7FD0D0` |
| NPP      | 時代力量      | `#F4D49A` |
| PFP      | 親民黨        | `#FF9A66` |
| TSU      | 台灣團結聯盟  | `#D6B890` |
| GPT      | 綠黨          | `#A3C75A` |
| IND      | 無黨籍        | `#B4BCC4` |
| EMPTY    | 尚未開票/平票 | `#D3D3D3` |

（註：可擴充其他政黨）

#### 2.2.3 縣市 Hover 互動

- 滑鼠移入縣市時，色塊加上 hover 高亮效果（例如加深或加上邊框）
- 顯示縣市名稱提示（tooltip），tooltip 內容顯示該區域前三名候選人（政黨點、姓名、得票率）

#### 2.2.4 縣市點擊互動 → 導向縣市詳情頁

點擊任一縣市後，將導向縣市詳情路由 `/county/:countyId`（實作行為），詳情頁含：

| 欄位         | 說明                                                                            |
| ------------ | ------------------------------------------------------------------------------- |
| 縣市名稱     | 顯示於頁面標題列                                                                |
| 開票進度     | 顯示目前開票百分比（例：`開票進度：100.0%`）                                    |
| 候選人結果表 | 顯示所有候選人（包含照片、政黨圖示、得票數、得票率、是否當選標示）              |
| 鄉鎮地圖     | 鄉鎮市區地圖顯示在右側（或版型內的地圖區塊），滑鼠 Hover 顯示該鄉鎮前三名候選人 |
| 行動按鈕     | （若仍需）可在詳情頁提供按鈕導回首頁或開啟完整歷史資料                          |

#### 2.2.5 縣市地圖圖例

地圖下方顯示本次選舉出現的政黨色彩對應圖例（與 2.3.4 鄉鎮地圖圖例格式一致）。`EMPTY`（`#D3D3D3`）應納入圖例，標示為「尚未開票 / 平票」。

### 2.3 縣市詳情頁（鄉鎮市區地圖）

#### 2.3.1 頁面結構

進入縣市詳情頁後，頁面分為左右兩欄：

左欄：選舉結果資訊

| 區塊           | 內容                                                                                    |
| -------------- | --------------------------------------------------------------------------------------- |
| 麵包屑導覽     | 例：`2022 縣市長選舉 > 南投縣`                                                          |
| 頁面標題       | 顯示「{縣市名稱}選情」                                                                  |
| 開票進度       | 顯示整體開票百分比                                                                      |
| 候選人結果表   | 顯示**所有候選人**的得票數與得票率                                                      |
| 候選人表格欄位 | 候選人照片、姓名、政黨圖示、是否當選標示（✓）、得票數、得票率；無照片時顯示預設灰色頭像 |
| 附註           | 說明得票率計算方式及資料來源（中選會或資料 API）                                        |

右欄：鄉鎮市區地圖

#### 2.3.2 鄉鎮市區地圖渲染

- 使用 **D3.js + TopoJSON** 繪製該縣市底下所有**鄉鎮市區**層級地圖
- 每個鄉鎮市區色塊以**該鄉鎮得票率最高政黨代表色**填充（純色，不使用漸層）

#### 2.3.3 鄉鎮市區 Hover 互動（僅 Hover，不點擊）

滑鼠移入鄉鎮市區時，顯示 **Tooltip**，內容包含：

| 欄位         | 說明                                                                       |
| ------------ | -------------------------------------------------------------------------- |
| 鄉鎮市區名稱 | 顯示於 Tooltip 標題                                                        |
| 前三名候選人 | 列出前三名候選人姓名、政黨圖示、得票率；若候選人不足三名，則顯示全部候選人 |

## 4. 資料結構需求

### 4.1 縣市層級資料

專案中 `Party` 型別與顏色以 `src/types/elections.ts` 為單一真實來源，示例如下：

```typescript
type Party =
  | "KMT" // 中國國民黨
  | "DPP" // 民主進步黨
  | "TPP" // 台灣民眾黨
  | "NPP" // 時代力量
  | "PFP" // 親民黨
  | "TSU" // 台灣團結聯盟
  | "GPT" // 綠黨
  | "IND" // 無黨籍
  | "EMPTY"; // 未分配或無資料

const PARTY_COLORS: Record<Party, string> = {
  KMT: "#5B6FA6",
  DPP: "#6FB98E",
  TPP: "#7FD0D0",
  NPP: "#F4D49A",
  PFP: "#FF9A66",
  TSU: "#D6B890",
  GPT: "#A3C75A",
  IND: "#B4BCC4",
  EMPTY: "#D3D3D3",
};

interface CountyResult {
  countyId: string; // 縣市代碼
  countyName: string; // 縣市名稱
  votingProgress: number; // 開票進度 (0–100, %)
  candidates: Candidate[]; // 所有候選人
}

interface Candidate {
  name: string; // 姓名
  party: Party; // 政黨
  votes: number; // 得票數
  voteRate: number; // 得票率 (%)
  elected: boolean; // 是否當選
  photoUrl?: string; // 候選人照片 URL（選填，無則顯示預設頭像）
}
```

### 4.2 鄉鎮市區層級資料

```typescript
interface TownshipResult {
  townshipId: string; // 鄉鎮市區代碼
  countyId: string; // 所屬縣市代碼
  townshipName: string; // 鄉鎮市區名稱
  votingProgress: number; // 開票進度 (0–100, %)
  candidates: Candidate[]; // 所有候選人
}
```

## 8. UI 設計規格：Bento Grid 版型

### 8.1 設計原則

採用 **Bento Grid** 版型作為全站佈局骨幹：

- 以 **12 欄格線** 為基礎，每格間距（gap）為 `16px`
- 每個功能區塊封裝為獨立「Bento Cell」，帶有白色背景、`border-radius: 12px`、`box-shadow: card`
- 視覺層次：大面積 Cell（地圖）主導版面，資訊 Cell（統計、圖例）輔助呈現
- Cell 內部 padding 統一為 `24px`

---

### 8.2 響應式行為

| 斷點                         | 版型行為                                             |
| ---------------------------- | ---------------------------------------------------- |
| `< 768px`（手機）            | 全欄單欄排列，地圖縮小至 `320px` 高，為 nice-to-have |
| `768px - 1279px`（平板）     | 地圖佔 `col-span-7`，資訊欄 `col-span-5`             |
| `≥ 1280px`（桌面，主要支援） | 依 8.2 / 8.3 版型全展開                              |
